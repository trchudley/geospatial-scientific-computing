# GUI GIS Software

This course teaches you to manipulate and plot geospatial data with Python. Programmatic interfaces are desirable for all the reasons we have already covered, such as scalability and replicability. However, there are still situations where you may wish to visualise and investigate data in a more casual manner. For this, we will use Geospatial Information Software (GIS), which allow you to manage geospatial data using a more tradiational Graphical User Interface (GUI) app.

Depending on the background with which you have come to this course, you may be familiar with QGIS software. If so, that is great and you probably won't learn much from this brief introduction. If you are not, and would like to be able to check the outputs your produce in this course by dragging-and-dropping them into a quick QUI interface, then this is the page for you.

## ArcGIS or QGIS?

There are effectively two dominant desktop GIS applications. The first is [ArcGIS Pro](https://www.esriuk.com/en-gb/arcgis/products/arcgis-pro/overview), a for-profit software that is widely used across industry. The second is [QGIS](https://www.qgis.org/), an open-source community software with extensive use across research, government, and NGOs.

My recommendation here is for QGIS (ArcGIS tutorials are available [here](https://doc.esri.com/en/arcgis-pro/latest/get-started/pro-quickstart-tutorials.html)). I am recommending QGIS because: (i) the free and open-source nature of the software will allow you to download and access it on your own computers as well as in the University teaching spaces; (ii) the software is available on MacOS and Linux operating systems as well as Windows (ArcGIS is Windows-only).
