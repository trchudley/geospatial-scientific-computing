# Using AI

The document is split into two: the fine print as required by the University, and a more human recommendation of how you might use AI within the programming aspects of this course.

## Permitted AI Use

This level of AI use allowed in this course is classified as **Category 3: Selective** according to University of Bristol AI Assessment guidelines.

For the formative and summative assessment you may use AI tools to aid your programming capability in the following ways:

 - **Exploring concepts**: You may use AI as a mechanism for researching and exploring relevant programming concepts and tools that may help you solve your research problems. You should always read the documentation of selected sources to make sure you understand what you are using and how to use it.
 - **Debugging**: You may use AI to comment on and debug your own code, to help you identify errors or improvements.
 - **Generating code**: You may use AI to directly generate code for your use. Be careful, as AI can generate code that will fail to run, generate incorrect results, or contain superflous or confusing content. You always remain responsible for the final output of any code you generate, and should be able to understand the code and apply it to other use cases.

You may **not** use AI tools to substantively help write any assessed prose (i.e. written text, as opposed to code). For writing prose, AI use aligns with the base level of use permitted in the [Academic Integrity pages](https://www.bristol.ac.uk/students/support/academic-advice/academic-integrity/). You may only use tools such as spelling and grammar checkers in this assignment, and their use should be limited to corrections of your own work rather than substantial re-writes or extended contributions.

Uses of tools outside of the stated limitations may incur academic misconduct penalties. You are not obligated to use AI tools in your work, and no additional credit is awarded for their use.

## Recommended AI Use

Using AI to write code is one of the most well-known use cases for large language model (LLM) AI models such as ChatGPT, Gemini, and Claude. It is now standard within industry and academia, and to ban AI use within this course would not be helpful in exposing you to realistic ways in which you may apply your programming skills in the future. I myself use LLMs to aid me in writing code for my own research: particularly for generating boilerplate code; finding ways to improve efficiencies; and to help write first drafts of function docstrings. However:

### Why you shouldn't use AI

Even though I will not stop you, I do not _recommend_ using AI throughout most of this course. The key takeaways of this course should not be the ability to write code: it should be the ability to understand foundational programming concepts and be able to apply them to geospatial problems. AI will be able to help you do this, but it cannot do this without the base level of understanding that this course aims to provide.

#### AI will not help you in the practical components of managing code and data.

Even in the first week of this course, you will be required to use new applications on your computer, navigate file structures using the command line, and have a solid understanding of directory structure to be able to find and use files. Later parts of the course require you to download novel data, and upload files to the internet. Raw text output from LLMs will not be able to do this for you: it requires an actual understanding of your local computing environment and how to engage with it.

#### Unlike LLM-generated text, LLM-generated is either correct or incorrect.

When LLMs write prose, it can write poorly and even contain incorrect information ("hallucinations"). However, it would still technically be readable as a piece of text. If LLMs make similar mistakes when writing code, there is no similar leeway: the code either works perfectly, or it will produce incorrect information or break entirely. Speaking from experience, trying to ask LLMs to resolve issues they have made can result in them making things worse, not better. If you are entirely reliant on AI to generate code, you will end up in a dead end and possibly in a worse position than when you started!

#### Debugging code with AI requires foundational programming knowledge.

Hence, the only way to be able to properly guide and use AI for programming purposes is by having the basic knowledge yourself. This allows you to ask the right questions, identify  basic issues or needs, be able to [communicate your requirements](https://cloud.google.com/discover/what-is-prompt-engineering), and be able to identify when LLMs are going wrong.

Unfortunately, **the only way to leearn this basic programming knowledge is by learning it the hard way!** That is what this course aims to help you do.

#### Easy access to LLMs is not guaranteed to remain in the future.

If you believe we are in [an AI bubble](https://en.wikipedia.org/wiki/AI_bubble), then it is likely that the current access we have to free or cheap state-of-the-art LLMs is not going to continue indefinitely. Understanding programming fundamentals will be even more necessary in this world, where costs are high, usage caps are common, or services are simply not available to you.
